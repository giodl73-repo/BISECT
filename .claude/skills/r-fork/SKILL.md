# r-fork

Use this skill before dispatching a pulse to an agent or before a long
implementation slice. The output is a single context file under the active wave.

## Usage

```text
/r-fork 04
```

## Procedure

1. Resolve the active wave from `context/waves/PHASES.md`.
2. Find `context/waves/{active}/pulses/{NN}+*.md`.
3. Read the pulse completely.
4. Extract `governing_roles` from frontmatter.
5. Resolve role notes:
   - If a role-specific file exists under `.roles/`, read it.
   - Otherwise record the role name and use the pulse text as the contract.
6. Write `context/waves/{active}/forks/pulse-{NN}.md`.

## Fork File Format

```markdown
# Fork Context: Pulse NN - Title

> Generated by /r-fork. Read this file completely before acting.

## Pulse Plan

{verbatim pulse file}

## Governing Roles

{role notes or "no local role file found"}

## Execution Contract

- Work in the repository root.
- Use `rg` for search.
- Use `apply_patch` for manual edits.
- Do not revert unrelated user changes.
- Check off pulse gates only after they are complete.
- Run the pulse validation commands before reporting done.
```

## Sync

If a fork is used as an execution receipt, copy completed checkbox state back to
the source pulse file after validation.
