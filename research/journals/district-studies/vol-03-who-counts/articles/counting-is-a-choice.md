# Counting Is a Choice

Equal population sounds like a complete rule. In practice, it is the start of a
rule. A redistricting system still has to decide which population count is being
equalized.

That choice is usually invisible because most maps use total population from the
decennial census. Total population is familiar, administratively available, and
deeply embedded in redistricting practice. But it is not the only possible
population vector. Voting-age population, citizen voting-age population, and
adjusted counts all ask the map to represent a slightly different public.

This issue begins there. It does not argue that one definition solves every
legal and policy problem. It asks what changes when the input changes.

The question matters because an algorithm cannot correct a population definition
on its own. If the population vector counts people at a facility address, the
map will balance that facility address. If it counts citizens of voting age, the
map will balance that smaller electorate. If it counts everyone, the map will
balance residents whether or not they can vote.

That is not a defect in the algorithm. It is a reminder that input choices are
governance choices.
