# The Trust Problem

The first objection to algorithmic redistricting is not technical. It is
trust-based: why should anyone believe a mapmaker that runs in software?

The right answer is not "trust us." The right answer is to show the
architecture. Bisect's baseline workflow is designed so that partisan data do
not enter the run. That means the default output cannot directly optimize for a
partisan result.

This is the point where practitioners need precision. The architecture does not
prove that every configuration is politically neutral. It proves something
more specific: the default workflow excludes partisan input, so any partisan
effect has to come from allowed criteria, data scope, or subsequent human
departures.
