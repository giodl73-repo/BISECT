# Input Exclusion Is the Point

The defense starts with a simple claim: the baseline workflow does not ingest
partisan data.

That matters because a mapmaker cannot directly optimize for something it never
sees. If the baseline does not take party into account, then the public
question changes. The issue is no longer whether the algorithm is secretly
targeting one side. The issue becomes whether the allowed inputs, seed choices,
and documented departures are good enough to justify the map that results.

That is a more defensible and more honest claim. It is also easier for a
commissioner or judge to understand, because it names the actual control points.
