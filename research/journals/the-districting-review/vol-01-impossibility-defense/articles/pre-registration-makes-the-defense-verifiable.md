# Pre-Registration Makes the Defense Verifiable

The input-exclusion defense becomes much stronger when the parameters are
pre-registered.

Pre-registration does not make the map correct by itself. What it does is bind
the run to a public description before the data arrives. That means a later
auditor can reproduce the baseline, compare departures, and verify that the
workflow followed the published rules.

This is a practical defense because it changes the evidentiary posture. The
question is no longer "could the mapmaker have changed the parameters?" The
question is "did the mapmaker actually change the published parameters, and can
we show it?"
