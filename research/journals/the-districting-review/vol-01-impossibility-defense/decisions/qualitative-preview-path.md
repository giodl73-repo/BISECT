# Qualitative Preview Path

## Decision

Use the **qualitative preview path** for Vol. 1.

The issue can be useful as public practitioner guidance before every support
artifact is closed, but only if it stays in the descriptive lane.

## What This Allows

- Use [B17] to explain bounded parameter variation.
- Use [R1] to explain pre-registration and stress testing.
- Use [B7] to show the baseline workflow is stable under the source scope.
- Use [A4] to explain commands, manifests, and hashes.
- Use [H0] to explain the court-facing posture.

## What This Blocks

- Universal impossibility claims.
- Claims that the baseline alone proves neutrality in every possible configuration.
- Claims that a manifest makes legal proof automatic.
- Claims that the algorithm itself forces adoption.

## Rationale

The issue is strongest when it tells practitioners how to describe the defense
accurately. That does not require a final law-review answer.
