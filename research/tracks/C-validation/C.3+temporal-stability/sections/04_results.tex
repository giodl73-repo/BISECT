\subsection{Primary Finding: Demographic Responsiveness Drives Cross-Census Variation}

Table~\ref{tab:county_disruption_real} shows county-level disruption for the bisect recursive algorithm between 2010 and 2020 census runs. Values are computed directly from run outputs in \texttt{outputs/2010/} and \texttt{outputs/2020/} using the 2010 and 2020 census tract demographics files.

\begin{remark}
Prior projection-based estimates (before census run data were available) showed $\sim$22\% disruption; the actual figure is 58--69\%. This discrepancy reflects a fundamental property of algorithmic redistricting: METIS re-optimises for the new census graph from scratch. When 21--42\% of census tracts are new (split or merged tracts), the resulting maps respond to demographic change rather than preserving prior boundaries. This is correct algorithmic behaviour, not instability.
\end{remark}

\subsection{The Performance-Stability Tradeoff}

Combining these temporal stability results with prior VRA performance findings:

\begin{itemize}
    \item \textbf{N-Way:} +4.3 percentage points better 2020 minority concentration, but -10 to -13 points worse 2020-2030 stability
    \item \textbf{Recursive:} Slightly lower 2020 performance, but substantially better long-term stability
\end{itemize}

For jurisdictions with comfortable VRA margins (e.g., multiple majority-minority districts achievable), recursive bisection's stability advantages likely outweigh marginal performance costs. For borderline cases (e.g., Alabama potentially gaining second minority-majority district), n-way's immediate performance gains may dominate.
