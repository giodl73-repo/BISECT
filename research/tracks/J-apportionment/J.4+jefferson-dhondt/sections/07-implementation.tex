\label{sec:implementation}

The \textsc{bisect-apportion} crate implements Jefferson/d'Hondt
using the priority formula $P_i^{\text{Jeff}}(s) = p_i/(s+1)$
(giving the $(s+1)$-th seat when already holding $s$ seats).
In the U.S.\ apportionment context, every entity starts with the constitutional
floor of one seat, so the first Jefferson priority for an additional seat is
$p_i/2$.

\begin{verbatim}
RoundingRule::Jefferson => p / (n + 1.0),
\end{verbatim}

\begin{verbatim}
let seats = apportionment_divisor(&populations, 435, RoundingRule::Jefferson);
\end{verbatim}

There is not yet a standalone \texttt{bisect apportion --method jefferson} or
\texttt{--method dhondt} CLI.

\subsection{D'Hondt Mode for PR Analysis}

The same library function can be used for d'Hondt-style proportional
representation analysis when the \texttt{populations} argument is interpreted as
party vote totals.  This is a library-level equivalence, not yet a polished
electoral-systems CLI.

\subsection{Test Suite}

\begin{itemize}
  \item Equal-population and 3:1 ratio synthetic examples cover the Jefferson path.
  \item A large-state-bias fixture compares Jefferson against Huntington-Hill.
  \item Shared divisor-method tests assert total-seat preservation and the
    one-seat constitutional floor.
  \item The package fixture
    \texttt{docs/examples/j-apportionment-evidence-packages/divisor-method-smoke}
    hash-binds a Jefferson/d'Hondt allocation and replays it through
    \texttt{bisect-apportion::divisor\_evidence}.
\end{itemize}
