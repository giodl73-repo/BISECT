We present \textsc{bisect-apportion}, a Rust crate implementing five
congressional apportionment methods --- Huntington-Hill, Webster, Adams,
Jefferson/d'Hondt, and Hamilton --- with two primary public functions:
\texttt{huntington\_hill()} and \texttt{apportionment\_divisor()}.
The central empirical claim is that the Huntington-Hill implementation
exactly reproduces the official 2020 Census Bureau congressional
apportionment (435 seats, 50 states), confirmed by string comparison
to the Census Bureau's April~26, 2021 published result:
zero discrepancies across all 50 states.

The priority computations use \texttt{f64} arithmetic.
For the seat counts and population values in U.S.\ apportionment
(populations $\leq 40$~million, seat counts $\leq 100$), priority comparisons
are exact because no population/seat product exceeds $4\times10^9$, well within
\texttt{f64}'s exact integer range of $2^{53} \approx 9\times10^{15}$.
SHA-256 seed verification is planned for Phase~2; currently the implementation
uses a deterministic seed formula without cryptographic verification.

The test suite comprises 95 tests across the crate at three levels:
L0 (unit tests on priority formulas and edge cases),
L1 (integration tests on synthetic examples with known correct answers),
and L2 (real-data regression tests against Census Bureau apportionments
for 2000, 2010, and 2020); 24 tests specifically cover apportionment
methods and paradox checking.
The Huntington-Hill L2 test achieves 100\% match rate (50/50 states)
for all three census years.
The crate also implements \texttt{check\_alabama\_paradox()},
which verifies that the Huntington-Hill allocation is house monotone
for Census data.

\textsc{bisect-apportion} is the apportionment component of the
\textsc{bisect} redistricting platform, providing the seat counts
that determine how many congressional districts each state draws
before the redistricting pipeline begins.
