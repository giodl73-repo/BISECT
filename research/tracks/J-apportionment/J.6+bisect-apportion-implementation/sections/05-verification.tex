\subsection{Verification Approach}

The current \textsc{bisect-apportion} implementation now ships a checked-in
2020 Census Table 1 regression fixture with SHA-256 source provenance.  The
package lives at
\texttt{docs/examples/j-apportionment-evidence-packages/2020-census-table01/}
and records the official Census XLSX URL, source hash, extracted 50-state
population and seat rows, verifier path, and replay command.  The verification
approach is:

\begin{enumerate}
  \item \textbf{Construct the canonical string}: Sort the 50 (state, seats)
    pairs alphabetically by state name.
    Concatenate as \texttt{"StateName:seats\textbackslash n"}
    for each pair.
  \item \textbf{Compare to reference}: The Census Bureau table
    \citep{census2020apportionment} should be processed into the same canonical
    format and compared string-by-string at test time.
  \item \textbf{Verify}: If the computed and reference strings match,
    the apportionment is correct relative to that fixture; otherwise the test fails.
\end{enumerate}

SHA-256 publication verification is now present for the 2020 source XLSX and
the extracted JSON fixture.  There is still no standalone \texttt{bisect
apportion --verify} CLI.

\subsection{The 2020 Verification Target}

The paper retains the Census Bureau's April~26, 2021 official apportionment
\citep{census2020apportionment} as the target reference.  The checked-in
fixture materializes the 50-state population table and official seat table,
computes \texttt{huntington\_hill()}, and compares every state to the official
allocation.

\subsection{2010 and 2000 Verification}

The same string-comparison protocol should be added for the 2010 and 2000
Census years.  Those rows remain future evidence, not current test coverage.

\subsection{Why Verification Matters for Redistricting}

In redistricting litigation and commission proceedings, the
apportionment step is often taken for granted.
Parties arguing over district maps rarely question the seat counts
assigned to each state.
But if the redistricting platform produces incorrect seat counts,
every downstream computation is wrong.

Verification provides:
\begin{itemize}
  \item \textbf{Reproducibility}: Any party can inspect the checked-in 2020
    package and rerun the verifier test.
  \item \textbf{Audit trail}: The package creates a documented
    link between the Census Bureau's official publication and the
    redistricting platform's input.
  \item \textbf{CI enforcement}: The verifier test ensures that code changes do
    not silently break the 2020 apportionment computation.
\end{itemize}
