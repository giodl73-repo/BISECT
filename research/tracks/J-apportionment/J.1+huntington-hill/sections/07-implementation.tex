\subsection{Priority Queue Implementation}

The \textsc{bisect-apportion} crate implements Huntington-Hill via
a max-priority queue using Rust's \texttt{BinaryHeap}.
The public function is:

\begin{verbatim}
pub fn huntington_hill(
    populations: &HashMap<String, u64>,
    total_seats: u32,
) -> HashMap<String, u32>
\end{verbatim}

Each state starts with the constitutional minimum of one seat.  The queue then
awards each remaining seat according to the \texttt{f64} priority
\[
  P_i(s_i) = \frac{p_i}{\sqrt{s_i(s_i+1)}}.
\]
Ties are broken deterministically by lexicographic state name order.

\subsection{Current Verification Boundary}

The crate has unit tests for the priority sequence, constitutional floor, seat
totals, and invalid house sizes.  It now also ships a hash-bound 2020 Census
Table 1 package at
\texttt{docs/examples/j-apportionment-evidence-packages/2020-census-table01/}.
The package records the official Census XLSX URL, source SHA-256, extracted
50-state apportionment rows, and a verifier test that recomputes
Huntington-Hill and matches the official 435-seat allocation.  A
\texttt{bisect apportion --verify} CLI and 2000/2010 packages remain future
hardening work.

\subsection{The 2020 Apportionment Target}

The 2020 Census Bureau apportionment is now the first checked-in reference
fixture.  Running the public \texttt{huntington\_hill()} function on the
extracted 50-state population table produces the official 435-seat allocation.

\subsection{Test Suite}

The current Huntington-Hill module tests:
\begin{itemize}
  \item a three-state, five-seat priority example;
  \item the minimum one-seat-per-state rule;
  \item total-seat preservation;
  \item a panic when total seats are fewer than states.
\end{itemize}
