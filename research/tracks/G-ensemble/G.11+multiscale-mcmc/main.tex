\begin{abstract}
\noindent
Standard \recom{}~\citep{deford2021} mixes slowly for large $k$: with
Texas ($k = 38$) or California ($k = 52$), each step restructures only two
of the $k$ districts, so $O(k)$ steps are required merely to touch every
district once, and tens of thousands of steps are needed for meaningful
plan-space exploration.
Multi-scale MCMC~\citep{autry2021} addresses this by interleaving two
chains at different spatial resolutions: a \emph{fine} chain (tract-level
\recom{}) that refines district boundaries locally, and a \emph{coarse}
chain (county-level \recom{}) that restructures large contiguous areas in
one step.
BISECT exposes the fixed-$\alpha$ version through
\texttt{bisect state --search multiscale}; the orchestration currently lives
in \texttt{bisect-cli}'s \texttt{run\_multiscale} wrapper, with shared seed,
hierarchy, configuration, and rebalance helpers in \texttt{bisect-multiscale}.
The key design insight is that county adjacency can be derived reproducibly from
the existing tract graph using GEOID prefix matching
(\texttt{county\_fips = geoid[:5]}): two counties are adjacent if and only
if any cross-county tract edge exists in the tract adjacency graph used by the
run.
This derivation requires no extra data files and no additional census
downloads.
The current CLI supports tract-to-county Option B without a block-group graph
and can exercise block-group fine resolutions when block-group adjacency is
supplied.
The evidence in this paper remains a short-run smoke comparison: Texas
lag-100 edge-cut autocorrelation improves in the recorded tables, but the
numbers should be read as implementation evidence rather than a portable
mixing theorem.
\end{abstract}
