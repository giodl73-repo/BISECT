# R1 Review — Moon Duchin
**Paper**: B.16 ConvergenceSweep: T=600 Statutory Seed Formula
**Score**: 3.0/4
**Verdict**: Major Revision

## Summary
B.16 certifies T=600 as the statutory stopping criterion for ConvergenceSweep via 50-state empirical data from B.7 and an extreme-value Gumbel model fit to the resulting convergence tails. The empirical contribution is concrete and valuable: Georgia's 511-seed tail is a real data point that establishes T=500 as insufficient and T=600 as adequate with an 89-seed margin. The theoretical infrastructure — Theorem 1 (Finite Effective Seed Space) and the Gumbel tail bound — has mathematical issues that need correction before publication. The algorithm specification and the SHA-256 seed formula are well-designed.

## Strengths
- The SHA-256 seed formula correctly distinguishes between unpredictability (randomness) and temporal neutrality (seed unknowable before census release). These are different properties; most redistricting proposals conflate them. The formula achieves temporal neutrality specifically, which is what the constitutional argument requires.
- Definition 2.2 (T-Stability) and Algorithm 1 are precisely specified. The strict less-than comparison in the algorithm — e_j < EC* (not <=) ensures the first minimum is retained in case of ties, preventing any ambiguity in the case of equal EC_norm values.
- The observation that T=500 was the early proposal and why it is insufficient is documented at the right level of rigor. The distinction between "the algorithm happens to return the correct plan at T=500" (true for Georgia) and "the algorithm can certify it has found the optimal plan at T=500" (false for Georgia) is mathematically precise and important for the statutory context.

## Concerns
1. **EC_norm definition inconsistency.** Section 2.1 defines EC_norm(Pi) using a recursive bisection tree formulation (sqrt(min(i, k_l - i)) at each level l). Algorithm 1 compares EC_norm(Pi_j) for full k-way partitions Pi_j produced by METIS. If METIS is called as a direct k-way partitioner (not via recursive bisection), there is no recursive tree and the "min(i, k_l - i)" denominator is undefined. The paper must specify a single unambiguous EC_norm definition for the ConvergenceSweep context — either (a) METIS is always called as a bisector in a recursive tree, in which case EC_norm sums over bisection levels, or (b) METIS produces a full k-way partition, in which case EC_norm = EC(Pi) / sqrt(k/2). Two different normalisations for two different call modes also raises the question of whether convergence tails tau are comparable across states using different modes.
2. **Theorem 1 proof sketch has an exponent error.** The paper states the number of distinct minimum-edge-cut k-partitions is O(n^{2k}), but the stated argument gives O(n^{2(k-1)}): a recursive bisection tree has k-1 internal nodes, each corresponding to a bisection cut, so an ordered (k-1)-tuple of cuts gives O(n^2)^{k-1} = O(n^{2(k-1)}). Additionally, the invocation of the Euler formula to bound the number of distinct cuts at O(n^2) conflates s-t cuts with balanced bisection cuts; these are different combinatorial objects. The theorem as stated contains two errors. Either fix the proof, or replace the theorem with the weaker but correct observation: "the METIS seed space is finite because each local optimum is a distinct partition of a finite set, so the sweep terminates in finite time" — which is all the paper needs.
3. **The Gumbel model's i.i.d. assumption is not justified.** The 50 convergence tails are outcomes from 50 structurally different graphs with different n, k, and geographic structure. The Gumbel extreme-value theorem applies to the maximum of n i.i.d. samples from the same distribution, which is not the case here. Fitting a Gumbel to these 50 heterogeneous observations and computing P(tau > 600) for "any US state" requires the implicit assumption that future US congressional graphs are drawn from the same distribution as the current 50 states — an empirical assumption that should be stated explicitly. A goodness-of-fit test (KS test) would help assess whether the Gumbel family is plausible for this data.
4. **The version string concatenation order in the SHA-256 formula is unspecified.** The statutory text in Section 5.4 specifies SHA-256(census_release_id || "DIA_SEED_V1") with byte concatenation. Section 4.3 discusses why the version string is needed for domain separation but does not specify whether the version string comes before or after the census_release_id in the hash input. While SHA-256 is not vulnerable to simple length-extension attacks here (the output is truncated to 31 bits via mod 2^{31}), the byte order should be specified unambiguously in the statute to prevent implementation divergence.
5. **No null hypothesis for T=600 adequacy.** The paper establishes T=600 is adequate empirically (89-seed margin over Georgia's 511) and theoretically (P(tau > 600) < 0.001 under Gumbel). But it does not articulate what it would mean for T=600 to be *inadequate*. What is the null hypothesis that T=600 is testing against? A precise null (e.g., "H0: the B.7 50-state dataset contains all worst-case states; the 2030 census will not produce a state with tau > 600") would clarify what the Gumbel model and empirical margin are jointly claiming.

## Required Changes
- **P1**: Provide a single unambiguous EC_norm definition for the ConvergenceSweep context in Algorithm 1. Specify whether METIS is called as a bisector (recursive tree) or as a direct k-way partitioner, and which normalisation formula applies in each case.
- **P1**: Correct Theorem 1: fix the exponent from O(n^{2k}) to O(n^{2(k-1)}) under the stated argument, and either correct the O(n^2) cuts-per-level argument or replace it with a simpler finiteness proof.
- **P1**: State explicitly that the Gumbel model treats the 50 convergence tails as approximately i.i.d. draws from the same distribution (an empirical assumption, not a consequence of extreme-value theory), add a KS goodness-of-fit test, and clarify that the statutory recommendation rests primarily on the empirical 89-seed margin.
- **P2**: Specify the byte-level concatenation order for the SHA-256 formula (which string comes first) in the statutory text.
