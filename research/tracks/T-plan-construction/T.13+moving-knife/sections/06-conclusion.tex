\section{Conclusion}
\label{sec:conclusion}

\mka{} gives BISECT a simple directional sweep alternative to graph cuts,
Voronoi assignment, and BFS growth. It is visually legible: for every candidate
angle, the reader can inspect the projection order, population prefix, two
halves, and worst-half Reock approximation.

\paragraph{Use today.}
Use \texttt{--structure moving-knife} when a run needs a geometry-first
baseline whose split choices are easy to replay. Treat the output as a
candidate structure plan that still needs the standard validation and scoring
pipeline.

\paragraph{Do not overclaim.}
The current implementation does not prove final-plan Reock optimality, does not
compute polygon-boundary Reock, and does not implement true Polsby-Popper
optimization inside \texttt{split\_subgraph\_mka}. It is a credible local
splitter, not yet a standalone certification system.

\paragraph{Next work.}
\begin{enumerate}
\item Archive per-node angle ledgers: projection order, prefix population,
  left/right scores, selected angle, and post-rebalance changes.
\item Add polygon-boundary Reock scoring for final plans and, if feasible, for
  candidate split ranking.
\item Implement real Polsby-Popper scoring only when perimeter data are present
  in the split routine.
\item Run same-metric NC/FL/WA packages against \metis{}, \cvdgeo{}, and
  \bfsg{}.
\item Add visual atlas panels that show each rejected angle and why the winner
  has the best worst-half enclosing-circle score.
\end{enumerate}

\paragraph{Summary.}
The useful contribution is not the phrase ``moving knife.'' It is the visible
ledger: show the sweep, show the prefix, show both enclosing-circle scores, and
show why that split was chosen. That is the standard this paper now sets for
\mka{}.
