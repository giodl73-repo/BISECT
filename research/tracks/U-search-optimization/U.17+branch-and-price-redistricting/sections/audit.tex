\section{Audit Artifacts}

Solved outputs should converge to the RPLAN/RCTX fixed point with certificate
and manifest sidecars. Column-generation-specific evidence remains in the run
context and solver report; the final RPLAN carries the assignment, not the
entire proof history.

The audit package should identify the column pool, selected columns, master
objective, fixture or instance id, and certificate profile. This keeps the
generic RPLAN assignment separate from the specialized column-generation
evidence needed to interpret exactness.

The public golden package can be checked either through the RPLAN certificate
verifier or through the BISECT manifest verifier.

The fixed point is deliberately two-layered. The RPLAN/RCTX pair fixes the
assignment and input graph; the branch-price report fixes the column-generation
evidence that led to the assignment. The algorithm lineage includes producer
\texttt{bisect-column}, method \texttt{branch-and-price}, report schema,
status, pricing rounds, generated column count, bounds, gap, and parameter
hash. Editing the selected columns or changing a formulation-only run into a
solved exact claim is therefore not an innocent prose change; it changes the
sidecar evidence that the package is expected to preserve.
