\section{Audit Artifacts}

The final-plan handoff is the RPLAN/RCTX fixed point. A branch-and-cut run can
emit a valid RPLAN plus solver report, manifest, and audit certificate. The
certificate verifies declared profile and lineage; it does not prove optimality
unless the solver report also carries the relevant proof or bound.

The audit package should retain both the plan artifact and the exact-method
claim artifact. RPLAN/RCTX records the final assignment and context, while the
solver report records whether separation, bounds, proof, timeout, or fallback
occurred. Dropping either side creates ambiguity: a valid plan without a solver
report is not an exact result, and a solver report without an auditable final
plan is not a reusable BISECT output.

The public golden package can be checked either through the RPLAN certificate
verifier or through the BISECT manifest verifier.

This is the fixed-point discipline that U.20 later generalizes. The plan and
context fix the district assignment and input universe; the manifest fixes the
method transcript and sidecar hashes; the ILP summary hash fixes the collection
of solve reports. If the LP model, proof-status field, fallback flag, or
summary JSON is edited after the fact, the hash chain changes. The verifier
does not need to trust the prose claim in this paper; it checks the declared
artifacts.
