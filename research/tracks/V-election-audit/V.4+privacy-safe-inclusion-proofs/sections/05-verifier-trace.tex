\section{Verifier Traceability}

The fixtures can be regenerated and checked directly:

\begin{quote}
\footnotesize
\begin{verbatim}
cargo run -p rcount-io --example privacy_inclusion_sketch_package
cargo run -p rcount-io --example choice_bearing_proof_package
cargo run -p rcount-cli -- verify \
  docs/examples/rcount-golden-packages/privacy-inclusion-sketch
cargo run -p rcount-cli -- verify \
  docs/examples/rcount-golden-packages/choice-bearing-proof
\end{verbatim}
\end{quote}

\begin{center}
\small
\begin{tabularx}{\textwidth}{lXX}
\toprule
Fixture & Expected result & Carrying check \\
\midrule
\texttt{privacy-inclusion-sketch} & transcript status \texttt{pass} & \texttt{proof\_privacy\_gate} passes for \texttt{proof:accepted-token-001}; \texttt{source\_hash\_match} also passes. \\
\texttt{choice-bearing-proof} & transcript status \texttt{fail} & \texttt{proof\_privacy\_gate} fails with \texttt{proof proof:accepted-token-001 exposes candidate selections}; \texttt{source\_hash\_match} still passes. \\
\bottomrule
\end{tabularx}
\end{center}

The negative transcript anchors the failure to the privacy gate:
\begin{quote}
\footnotesize
\begin{verbatim}
{
  "equation_id": "proof_privacy_gate",
  "status": "fail",
  "error": "proof proof:accepted-token-001 exposes candidate selections"
}
\end{verbatim}
\end{quote}

That separation matters. The package can be well-formed, hash-bound, and
arithmetically coherent while still being unacceptable for public voter-facing
proof use.
