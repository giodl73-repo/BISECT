\begin{abstract}
Public election-count evidence is fragile unless the bytes being verified are
named, hashed, and replayed. This paper describes RCOUNT's tamper-evidence
layer: source hashes for raw public inputs, package hashes for normalized
records, batch manifests for smaller accounting units, and transcripts that
separate hash failures from arithmetic failures. Hashing does not certify an
election or prove tabulator correctness. It makes later changes visible: if a
source file changes after publication, a verifier can detect the mismatch even
when summary arithmetic still passes.
\end{abstract}
