Computing the exact minimum bounding circle (MBC) of a polygon can be reduced to
computing the smallest enclosing disk (SED) of the relevant boundary point set,
solvable in expected linear time by Welzl's randomised algorithm~\citep{welzl1991smallest}.
That algorithm is the correct reference for exact-MBC Reock, but it is not the
same as every Reock-like score exposed by \textsc{bisect}.

The production implementation in
\texttt{crates/bisect-analysis/src/compactness.rs} uses a
\textbf{centroid-plus-max-radius approximation}: the circle centre is taken as
the polygon centroid, and the radius is the maximum distance from the centroid
to any exterior boundary coordinate.  This approximation matches the convention
used in the Python redistricting pipeline that \textsc{bisect} supersedes,
ensuring comparability of Reock scores across pipeline versions.

\begin{remark}[Future work]
Exact polygon MBC remains future work for the compactness report.
The centroid-plus-max-radius approximation yields a circle that contains the polygon
but may be larger than the true MBC (the approximation errs toward over-coverage,
producing Reock values that are conservative---i.e., slightly lower than true-MBC Reock).
Practitioners requiring exact MBC-based Reock (e.g., for reproduction against
other redistricting software) should note this approximation and should not cite
\texttt{bisect label-analyze} Reock values as exact-MBC scores.
\end{remark}

Test invariants (L0):
\begin{itemize}
  \item Reock of a circle is $\leq 1.0$ (the approximation may give Reock $< 1$
    for a circle if the centroid is not the true MBC centre; for a centred circle
    the approximation is exact).
  \item Approximation circle contains all input boundary points: for all $p$ on the
    boundary, $\|p - \text{centroid}\| \leq r + 10^{-9}$.
  \item Approximation circle area $\geq$ polygon area for any convex polygon.
\end{itemize}
