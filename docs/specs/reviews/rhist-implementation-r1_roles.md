## Decision

Proceed fixture-first. Do not extract `rctx-core` yet. Do not migrate RCOUNT
lineage yet. The tiny `rhist-core` verifier, `rhist-io` package loader, and
`rcount-rhist` mapping bridge now exist for L0/L1 fixtures. Canonical package
hashing also exists in `rhist-core`/`rhist-io`, with a fixture regeneration
command. A three-cycle RHIST fixture now exists; next, add either a tiny CLI
verification command or a real-source pressure fixture.

Update: `rhist-cli verify` now exists, and
`docs/fixtures/rhist/real-ri-tract-unchanged` provides the first real-source
pressure fixture. Next is consumer integration: RCOUNT should reference RHIST
package hashes rather than expanding independent cross-cycle history semantics.
