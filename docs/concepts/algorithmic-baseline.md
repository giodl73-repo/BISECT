# Algorithmic Baseline

## Short Version

Use **algorithmic baseline** as the public term for Bisect in policy-facing
contexts. It is more precise than "redistricting AI" and less legally loaded
than "mandatory algorithmic redistricting."

## What It Means

An algorithmic baseline is a reproducible reference plan generated from public
inputs, published code, and documented parameters. It can be used for comparison,
commission workflow, court remedy design, or proposed statutory language.

It does not mean:

- Every legal question is solved.
- Every jurisdiction must use the same final map.
- Outcomes are guaranteed to be politically neutral.
- Courts will accept every statistical or audit claim without review.

## Public Language

| Use | Avoid until reviewed |
|-----|----------------------|
| reproducible baseline | proof of fairness |
| input-exclusion defense | impossibility defense |
| candidate federal rule | the federal rule |
| reviewed recommendation | inevitable conclusion |
| source-chain audit | trustless democracy |

## Review Posture

Before public locked copy, route baseline claims through:

- MERIDIAN for algorithm correctness
- CONTOUR for data provenance
- BENCHMARK for reproducibility
- SCALE for statistics
- BOUNDARY and WARD for legal scope
- COVENANT for audit and evidence standards
